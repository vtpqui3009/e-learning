## What is Automated Testing?

Automated testing means writing code that tests other code. Instead of manually clicking buttons and checking results, you write test scripts that do it automatically.

**Manual Testing** (what you might be doing now):
1. Open the application
2. Click a button
3. Enter some data
4. Check if the result is correct
5. Repeat for every feature... every time!

**Automated Testing** (what you'll learn):
1. Write a test script once
2. Run it automatically
3. Get results in seconds
4. Run thousands of tests in minutes

## Types of Tests

### 1. Unit Tests
Test individual methods or classes in isolation.

**Example**: Testing if a calculator's Add method works correctly.

### 2. Integration Tests
Test how different parts of your application work together.

**Example**: Testing if your code can save data to a database and retrieve it correctly.

### 3. End-to-End (E2E) Tests
Test the entire application from start to finish.

**Example**: Testing a complete user registration flow from opening the app to receiving a confirmation email.

### 4. API Tests
Test web services and APIs.

**Example**: Testing if your REST API returns the correct data when you send a request.

## What Makes a Good Test?

A good automated test should be:

1. **Fast**: Runs in milliseconds, not minutes
2. **Isolated**: Doesn't depend on other tests
3. **Repeatable**: Same result every time
4. **Self-checking**: Automatically determines pass/fail
5. **Clear**: Easy to understand what it's testing

## Common Testing Terms

- **Test Case**: A single test that checks one thing
- **Test Suite**: A collection of test cases
- **Assertion**: A check that something is true
- **Test Runner**: Tool that executes tests
- **Test Framework**: Library that helps write tests (xUnit, NUnit, MSTest)
- **Mock**: A fake object used in testing
- **Arrange-Act-Assert**: Pattern for structuring tests
