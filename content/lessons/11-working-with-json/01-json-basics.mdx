# JSON Basics

JSON (JavaScript Object Notation) is the most common format for exchanging data between applications, especially web APIs. Learning JSON is essential for working with modern C# applications.

## What is JSON?

JSON is a text format for representing structured data. It's easy for humans to read and easy for computers to parse.

<CodeEditor
  initialCode={`using System;

class Program
{
    static void Main()
    {
        // This is what JSON looks like
        string jsonExample = @"{
            ""name"": ""Alice"",
            ""age"": 30,
            ""email"": ""alice@example.com"",
            ""isActive"": true
        }";

        Console.WriteLine("JSON Example:");
        Console.WriteLine(jsonExample);

        Console.WriteLine("\nJSON is just text!");
        Console.WriteLine($"Length: {jsonExample.Length} characters");
    }
}`}
/>

## Why Use JSON?

| Reason | Description |
|--------|-------------|
| **Universal** | Works with any programming language |
| **Human-Readable** | Easy to read and understand |
| **Lightweight** | Smaller than XML, faster to transmit |
| **Web Standard** | Default format for REST APIs |

## JSON vs C# Classes

JSON objects look similar to C# classes:

<CodeEditor
  initialCode={`using System;

// C# Class
class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    public string Email { get; set; }
    public bool IsActive { get; set; }
}

class Program
{
    static void Main()
    {
        // C# Object
        var person = new Person
        {
            Name = "Alice",
            Age = 30,
            Email = "alice@example.com",
            IsActive = true
        };

        Console.WriteLine("=== C# Object ===");
        Console.WriteLine($"Name: {person.Name}");
        Console.WriteLine($"Age: {person.Age}");
        Console.WriteLine($"Email: {person.Email}");
        Console.WriteLine($"Active: {person.IsActive}");

        // Equivalent JSON (as text)
        string jsonPerson = @"{
            ""Name"": ""Alice"",
            ""Age"": 30,
            ""Email"": ""alice@example.com"",
            ""IsActive"": true
        }";

        Console.WriteLine("\n=== JSON Representation ===");
        Console.WriteLine(jsonPerson);

        Console.WriteLine("\nThey represent the same data!");
        Console.WriteLine("C# = Strongly typed objects");
        Console.WriteLine("JSON = Text representation of objects");
    }
}`}
/>

## JSON Syntax Rules

⚠️ **Important Rules:**

1. **Double Quotes Only**: Use `"` not `'`
   - ✅ `{"name": "Alice"}`
   - ❌ `{'name': 'Alice'}`

2. **No Trailing Commas**: Last item has no comma
   - ✅ `{"a": 1, "b": 2}`
   - ❌ `{"a": 1, "b": 2,}`

3. **Keys Must Be Strings**: Property names in quotes
   - ✅ `{"name": "Alice"}`
   - ❌ `{name: "Alice"}`

4. **Case Sensitive**: `Name` ≠ `name`

## Common JSON Use Cases

### 1. REST APIs
```json
GET /api/users/123

Response:
{
  "id": 123,
  "name": "Alice",
  "email": "alice@example.com"
}
```

### 2. Configuration Files
```json
appsettings.json:
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=myapp"
  },
  "Logging": {
    "LogLevel": "Information"
  }
}
```

### 3. Data Storage
- NoSQL databases (MongoDB, CosmosDB)
- Cache systems (Redis)
- Message queues

### 4. Frontend ↔ Backend Communication
- Web applications send/receive JSON
- Mobile apps communicate via JSON APIs
- Microservices exchange JSON data
